<Callout type="info" title="What is tokenfactory-skills?">
  **tokenfactory-skills** is the official AI editor Skill plugin from TokenFactory. Skills are a lightweight extension protocol that enables AI coding assistants — such as Claude Code, Codex, OpenClaw, Cursor, Windsurf, and Cline — to **call TokenFactory endpoints directly from within the editor**, performing model queries, token management, balance checks, and answering TokenFactory deployment, configuration, and API usage questions without ever leaving your coding environment.

  - GitHub Repository: [github.com/fyinfor/skills](https://github.com/fyinfor/skills)

  It now also works inside OpenClaw, so you can run `/tokenfactory` commands for models, tokens, groups, and balance, and ask TokenFactory usage questions without leaving your agent workflow.
</Callout>

## Why Use Skills

During everyday AI development, developers frequently switch between their editor and the TokenFactory management dashboard — checking available models, creating API tokens, inspecting balance, adjusting group quotas. These context switches disrupt coding flow.

**What tokenfactory-skills solves:**

- **Zero context-switching**: Issue natural language commands directly in Claude Code, Codex, OpenClaw, or other AI IDEs — no browser or admin panel needed
- **Built-in help**: Ask about TokenFactory deployment, configuration, groups, and API call formats directly from your coding workflow
- **Security-first design**: Keys are never shown in plain text; token copy and injection use secure channels (`sk-` prefixed keys never appear in terminal output or logs)
- **Instant setup**: A single `npx` command handles installation; runtime auto-detection (Bun / Node.js / Deno)
- **Cross-editor compatibility**: Works with any AI editor or coding assistant that implements the Skills protocol

## Commands Reference

Below is the complete list of commands provided by tokenfactory-skills. These commands interact with the TokenFactory REST API to query and manage resources on your TokenFactory instance.

### Query Commands

| Command | Description | Use Case |
|---------|-------------|----------|
| `/tokenfactory models` | List available AI models | View all models configured on your TokenFactory instance (OpenAI, Claude, Gemini, etc.) |
| `/tokenfactory groups` | List user groups | View your account's groups with quota and rate multiplier settings |
| `/tokenfactory balance` | Show account balance | Real-time query of current user's balance and usage |

### Token Management Commands

| Command | Description | Use Case |
|---------|-------------|----------|
| `/tokenfactory tokens` | List API tokens | View all created tokens (keys shown masked, e.g. `sk-reHR**********OspA`) |
| `/tokenfactory create-token <name> [--group=xxx]` | Create Token Factory token | Generate independent API keys for different apps or projects |
| `/tokenfactory switch-group <token_id> <group>` | Change token's group | Modify token group to adjust model access permissions and rate multipliers |
| `/tokenfactory copy-token <token_id>` | Copy key to clipboard | Securely copy the real key to system clipboard (key never displayed in terminal) |
| `/tokenfactory apply-token <token_id> <file>` | Inject key into config file | Securely write token key into project config files (e.g. `.env`) via placeholder replacement |

### Help Command

| Command | Description | Use Case |
|---------|-------------|----------|
| `/tokenfactory help <question>` | Ask about TokenFactory | Get help on TokenFactory deployment, configuration, API call formats, and more |

## Installation & Configuration

<Steps>
  <Step>
    ### Install tokenfactory-skills

    One-line installation via `npx`:

    ```bash
    npx skills add https://github.com/fyinfor/skills --skill tokenfactory
    ```

    This pulls the latest Skill definition from the GitHub repository and installs it into your editor environment.
  </Step>
  <Step>
    ### Set Environment Variables

    tokenfactory-skills requires three environment variables to connect to your TokenFactory instance. Add them to your shell profile:

    ```bash
    # ~/.bashrc or ~/.zshrc
    export TOKENFACTORY_BASE_URL=https://<your-tokenfactory-domain>
    export TOKENFACTORY_ACCESS_TOKEN=your-access-token
    export TOKENFACTORY_USER_ID=1
    ```

    | Variable | Description | Example |
    |----------|-------------|---------|
    | `TOKENFACTORY_BASE_URL` | TokenFactory service URL | `https://api.example.com` |
    | `TOKENFACTORY_ACCESS_TOKEN` | Admin or user access token | From "Personal Settings → Account → Security → System Access Token" |
    | `TOKENFACTORY_USER_ID` | User ID | `1` (admin default is 1) |

    You can also create a `.env` file in your project root. Make sure `.env` is listed in `.gitignore` to prevent key leaks.
  </Step>
  <Step>
    ### Start Using

    Once installed, type `/tokenfactory` commands in any Skills-compatible AI editor. For example:

    - Type `/tokenfactory models` to list available models
    - Type `/tokenfactory balance` to check your current balance
    - Type `/tokenfactory create-token my-app` to create a new token named `my-app`
    - Type `/tokenfactory help how do I configure groups and tokens?` to ask TokenFactory usage questions directly
    - In OpenClaw, you can also run `/tokenfactory models`, `/tokenfactory balance`, and token commands directly in your agent workflow
  </Step>
</Steps>

## Security

tokenfactory-skills has multiple layers of security built in to protect your API keys:

- Token listings only show **masked keys** (e.g. `sk-reHR**********OspA`) — real keys never appear in terminal output
- `copy-token` copies the key to the system clipboard only, never displaying it in any output
- `apply-token` uses placeholder replacement to inject keys, never exposing raw key content
- All `sk-` prefixed key values are strictly protected throughout the entire execution flow
